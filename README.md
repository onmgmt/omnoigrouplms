# Omnoi Academy — ระบบ LMS ภายในองค์กร

ระบบเรียนรู้ออนไลน์ภายในของ Omnoi Group สำหรับพัฒนาทักษะพนักงาน แยกตามแผนก พร้อมระบบ Level คอร์ส (ต้องผ่านคอร์สก่อนหน้าก่อนถึงจะเรียนต่อได้) และบันทึกกิจกรรมผู้ใช้ (audit log)

- **เว็บแอป:** หน้าเดียว HTML/CSS/JS (`index.html`) เรียก API ฝั่งเซิร์ฟเวอร์
- **Backend:** PHP ล้วน (ไม่มี framework/Composer) ในโฟลเดอร์ `api/`
- **ฐานข้อมูล:** MySQL / MariaDB (ไฟล์ `schema.sql`)
- **โฮสต์เว็บ:** HostNeverDie (DirectAdmin + MySQL + phpMyAdmin) ที่ `lms.omnoigroup-it.com`
- **วิดีโอ/เอกสาร:** ฝังจาก Google Drive (แชร์ลิงก์แบบ "ทุกคนที่มีลิงก์ - ดูได้" แล้ววางลิงก์ในระบบ)
- **เวอร์ชัน:** จัดการด้วย Git / GitHub + auto-deploy ผ่าน GitHub Actions (FTP)

> สถานะปัจจุบัน: **ระบบจริง (production-ready)** — ผู้ใช้/แผนก/คอร์ส/คะแนน/log เก็บใน MySQL กลาง ไม่ใช่ localStorage ของเบราว์เซอร์อีกต่อไป รองรับผู้ใช้หลายคนพร้อมกันจริง

## คุณสมบัติ

1. **เข้าสู่ระบบ** ด้วย username + password จริง (เก็บรหัสผ่านแบบ hash ในฐานข้อมูล ไม่ใช่ plaintext) พร้อมล็อกบัญชีชั่วคราวหลัง login ผิดหลายครั้งติดกัน
2. **แผนก (Department)** สร้าง/แก้ไขได้จากเมนูแอดมิน — พนักงาน 1 คนสังกัดได้หลายแผนกพร้อมกัน
3. **คอร์สอยู่ภายในแผนก** และมี **Level** — คอร์ส Level ถัดไปจะถูกล็อกจนกว่าพนักงานจะสอบผ่านคอร์ส Level ก่อนหน้าในแผนกเดียวกัน (บังคับจริงที่ฝั่งเซิร์ฟเวอร์ ไม่ใช่แค่ซ่อนปุ่มฝั่งหน้าเว็บ)
4. **ฝังวิดีโอบทเรียนจาก Google Drive** ในแต่ละบท — พนักงานกดยืนยัน "ดูจบแล้ว" เองหลังดูวิดีโอ เพื่อปลดล็อกบทถัดไปทีละบท (ไม่มีระบบบล็อกกรอ/บล็อกข้ามวิดีโอ)
5. **เอกสารประกอบการเรียนต่อบทเรียน** — ดาวน์โหลดได้จาก Google Drive เช่นกัน ปลดล็อกให้ดาวน์โหลดหลังพนักงานกด "ดูจบแล้ว" ของบทนั้น
6. **แบบทดสอบท้ายคอร์ส + บันทึกคะแนน** รายบุคคล (เกณฑ์ผ่าน 80% เก็บคะแนนสูงสุด/จำนวนครั้งสอบ) — ผ่านแล้วปลดล็อกคอร์ส Level ถัดไปอัตโนมัติ พร้อม **หน้าแอดมินจัดการคำถามแบบทดสอบ** (เพิ่ม/แก้ไข/ลบคำถามและตัวเลือกผ่านหน้าเว็บได้โดยตรง ไม่ต้องใช้ phpMyAdmin)
7. **แก้ไขคอร์สที่สร้างไปแล้ว** — แก้ชื่อ คำอธิบาย ไอคอน สี แผนก Level และเพิ่ม/แก้ไข/ลบบทเรียนของคอร์สเดิมได้จากหน้า "จัดการคอร์ส" (บทเรียนเดิมที่ยังอยู่จะไม่เสียประวัติ "ดูจบแล้ว" ของพนักงาน — เสียเฉพาะบทที่ถูกลบออกจริงเท่านั้น)
8. **ซ่อน/เผยแพร่คอร์ส (Publish)** — คอร์สที่ยังไม่พร้อม (ฉบับร่าง) ซ่อนจากพนักงานได้ทันทีด้วยปุ่มเดียวจากหน้า "จัดการคอร์ส" โดยไม่ต้องลบข้อมูล เผยแพร่กลับได้เมื่อพร้อม (บังคับที่ฝั่งเซิร์ฟเวอร์ — เรียก API ตรงๆ ข้ามคอร์สที่ซ่อนอยู่ไม่ได้เช่นกัน)
9. **จัดการผู้ใช้ (แอดมิน)** — สร้าง username/password ให้พนักงานแต่ละคน กำหนดแผนกที่สังกัดได้หลายแผนก เปิด/ปิดใช้งานบัญชี รีเซ็ตรหัสผ่าน
10. **รายงานคะแนนรายบุคคล + ประวัติการใช้งาน (Activity Log)** — ดูได้ว่าใคร login เมื่อไร เปิดวิดีโอบทไหน โหลดเอกสารไหน คะแนนสอบเท่าไร และเรียนจบคอร์สไหนไปแล้วบ้าง กรองตามผู้ใช้/ประเภทเหตุการณ์/ช่วงวันที่ได้

## รันในเครื่อง (local, สำหรับทดสอบก่อน deploy)

ต้องมี PHP + MySQL/MariaDB ในเครื่อง:

```bash
# 1) สร้างฐานข้อมูลว่างในเครื่อง แล้ว import schema.sql
mysql -u root -e "CREATE DATABASE lms_test CHARACTER SET utf8mb4"
mysql -u root lms_test < schema.sql

# 2) แก้ webapp/api/config.php ให้ชี้ไปที่ฐานข้อมูลข้างต้น (DB_HOST/DB_NAME/DB_USER/DB_PASS)

# 3) รันเว็บด้วย PHP built-in server จากโฟลเดอร์ webapp/
php -S localhost:8080
# แล้วเปิด http://localhost:8080
```

> หมายเหตุ: PHP built-in server (`php -S`) ไม่อ่านไฟล์ `.htaccess` (เป็นฟีเจอร์ของ Apache เท่านั้น) — ตอน deploy จริงบน HostNeverDie (Apache) ไฟล์ `api/.htaccess` จะทำงานป้องกัน `config.php` ตามปกติ

## ผูกวิดีโอ + เอกสารประกอบ Google Drive

1. อัปโหลดวิดีโอ/เอกสารบทเรียนขึ้น Google Drive (โฟลเดอร์ของบริษัท/ทีม)
2. คลิกขวาที่ไฟล์ → **แชร์** → เปลี่ยนสิทธิ์เป็น **"ทุกคนที่มีลิงก์ - ดูได้ (Viewer)"** (จำเป็น มิฉะนั้นพนักงานจะเปิด/ดาวน์โหลดไม่ได้)
3. คัดลอก **ลิงก์แชร์** เช่น `https://drive.google.com/file/d/<FILE_ID>/view?usp=sharing`
4. เข้าสู่ระบบเป็น `admin` → เมนู **จัดการคอร์ส** → เลือกแผนก กำหนด Level → วางลิงก์ (หรือแค่ `<FILE_ID>` ก็ได้) ในช่องวิดีโอ และ/หรือช่องเอกสารประกอบของแต่ละบท — ระบบจะดึง File ID ออกจากลิงก์ให้อัตโนมัติที่ฝั่งเซิร์ฟเวอร์
5. บทที่ใส่ลิงก์วิดีโอแล้วจะแสดงวิดีโอ Google Drive จริงในหน้าเรียน ส่วนบทที่เว้นว่างจะขึ้นข้อความ **"ยังไม่ได้ผูกวิดีโอ"**
6. เอกสารประกอบ (ถ้ามี) จะขึ้นเป็นปุ่ม **"⬇ ดาวน์โหลดเอกสาร"** ในหน้าเล่นวิดีโอและหน้าคอร์ส แต่จะล็อกไว้จนกว่าพนักงานจะกดยืนยัน "ดูจบแล้ว" ของบทนั้นก่อน

> วิดีโอ Google Drive ฝังผ่าน iframe ของ Google เอง จึงใช้ปุ่มเล่น/หยุด/กรอวิดีโอตามปกติของ Google Drive ได้ — เว็บไม่สามารถบล็อกการกรอหรือรู้ว่าดูจบจริงหรือไม่ ระบบจึงใช้วิธี **ให้พนักงานกดปุ่ม "✓ ทำเครื่องหมายว่าดูจบแล้ว" เอง** เพื่อปลดล็อกบทถัดไป เอกสารประกอบ และแบบทดสอบ (แบบ trust-based เหมือนเดิม)

## ระบบ Level คอร์ส (ต้องผ่านคอร์สก่อนหน้าก่อน)

- แต่ละคอร์สมีเลข **Level** ภายในแผนกของตัวเอง (Level นับแยกต่อแผนก ไม่ปนกัน)
- คอร์ส Level 1 ปลดล็อกเสมอสำหรับพนักงานที่สังกัดแผนกนั้น
- คอร์ส Level N (N > 1) จะ **ล็อก** จนกว่าพนักงานจะสอบผ่านคอร์ส Level N-1 **ในแผนกเดียวกัน** (คะแนน ≥ 80%)
- ตอนสร้างคอร์สในเมนู "จัดการคอร์ส" ถ้าเว้นช่อง Level ว่างไว้ ระบบจะกำหนด Level ให้อัตโนมัติเป็นลำดับถัดไปของแผนกนั้น
- กติกานี้ตรวจสอบที่ **ฝั่งเซิร์ฟเวอร์เสมอ** (ไม่ใช่แค่ซ่อนปุ่มฝั่งหน้าเว็บ) — เรียก API ตรงๆ ข้ามคอร์สที่ล็อกอยู่ไม่ได้

## Deploy ขึ้น HostNeverDie (`lms.omnoigroup-it.com`)

### 1) สร้างฐานข้อมูล MySQL

1. เข้า DirectAdmin ของ HostNeverDie → เมนู **MySQL Management** → **Create new Database**
2. ตั้งชื่อฐานข้อมูล + ผู้ใช้ + รหัสผ่าน (DirectAdmin มักเติม prefix ชื่อบัญชีให้อัตโนมัติ เช่น `user_lmsdb`) — จดค่าทั้ง 4 นี้ไว้ (host มักเป็น `localhost`)

### 2) Import schema.sql

1. จากหน้า MySQL Management ใน DirectAdmin กด **phpMyAdmin** เพื่อเข้าจัดการฐานข้อมูลที่เพิ่งสร้าง
2. เลือกฐานข้อมูล → แท็บ **Import** → เลือกไฟล์ `schema.sql` จากโฟลเดอร์นี้ → กด Go
3. ตรวจว่ามี 10 ตารางถูกสร้างขึ้น (users, departments, user_departments, courses, lessons, quiz_questions, lesson_progress, quiz_attempts, course_completions, activity_logs) พร้อมข้อมูลตัวอย่าง

### 3) ตั้งค่า config.php

แก้ไฟล์ `api/config.php` ให้ตรงกับค่าฐานข้อมูลจริงจากขั้นตอนที่ 1:

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'ชื่อฐานข้อมูลจริง');
define('DB_USER', 'ชื่อผู้ใช้ฐานข้อมูลจริง');
define('DB_PASS', 'รหัสผ่านฐานข้อมูลจริง');
```

`DEBUG_MODE` ต้องเป็น `false` เสมอบนเซิร์ฟเวอร์จริง (กันไม่ให้ข้อความ error ของ PHP/SQL หลุดออกไปให้ผู้ใช้เห็น)

### 4) เลือกเวอร์ชัน PHP

ใน DirectAdmin → **PHP Selector** (หรือเมนูใกล้เคียง) เลือก **PHP 8.x** ให้กับ subdomain/โดเมนที่ใช้

### 5) อัปโหลดไฟล์

1. ใน DirectAdmin สร้าง **subdomain** `lms.omnoigroup-it.com` (ถ้ายังไม่มี)
2. อัปโหลดทั้งโฟลเดอร์นี้ (`index.html`, `logo.png`, ทั้งโฟลเดอร์ `api/` — **ไม่ต้อง** อัป `schema.sql` ขึ้นเว็บ เพราะ import ผ่าน phpMyAdmin ไปแล้ว) เข้าไปยัง document root ของ subdomain (เช่น `domains/omnoigroup-it.com/public_html/lms/`) ผ่าน File Manager, FTP, หรือ GitHub Actions (ดูหัวข้อ Auto-deploy ด้านล่าง)
3. เปิดใช้ **SSL ฟรี (Let's Encrypt)** ให้ subdomain

### 6) ทดสอบและตั้งค่าความปลอดภัยเริ่มต้น

1. เปิด `https://lms.omnoigroup-it.com` → login ด้วย `admin` / `admin`
2. ไปที่เมนู **จัดการผู้ใช้** → แก้ไขบัญชี `admin` → ตั้งรหัสผ่านใหม่ทันที
3. ลบ/ปิดใช้งานบัญชีตัวอย่าง (`somchai`, `nattaya`, `weera`, `ploy`) ถ้าไม่ได้ใช้จริง หรือแก้ไขให้เป็นพนักงานจริง
4. ตรวจสอบว่าเปิด `https://lms.omnoigroup-it.com/api/config.php` ตรงๆ แล้วต้องขึ้น **403 Forbidden** (ไฟล์ `.htaccess` ทำงานถูกต้อง) — ถ้าไม่ขึ้น 403 ให้ตรวจสอบว่า Apache เปิดใช้ `.htaccess` (`AllowOverride`) ให้กับโฟลเดอร์นี้แล้ว แจ้งฝ่าย support ของ HostNeverDie ได้หากต้องเปิดสิทธิ์เพิ่ม

## Auto-deploy: GitHub → HostNeverDie (FTP)

repo นี้มี GitHub Actions (`.github/workflows/deploy.yml`) ที่จะ **อัปไฟล์ขึ้น HostNeverDie ผ่าน FTP ให้อัตโนมัติทุกครั้งที่ push ขึ้น branch `main`** (และกดรันเองได้ที่แท็บ Actions) — จะอัปทั้งโฟลเดอร์ `webapp/` รวมถึง `api/` ให้อัตโนมัติ

ตั้งค่าครั้งเดียว — เพิ่ม **Secrets** ใน GitHub repo:
`Settings → Secrets and variables → Actions → New repository secret`

| ชื่อ Secret | ค่า |
|-------------|-----|
| `FTP_SERVER` | `ftp.omnoigroup-it.com` (หรือ IP เซิร์ฟเวอร์) |
| `FTP_USERNAME` | ชื่อบัญชี FTP |
| `FTP_PASSWORD` | รหัสผ่าน FTP |

จากนั้นทุก `git push` → ดูสถานะการ deploy ได้ที่แท็บ **Actions**

หมายเหตุ:
- ปรับ `server-dir` ในไฟล์ workflow ให้ตรงกับโฟลเดอร์ปลายทาง — ถ้าบัญชี FTP จำกัดอยู่ในโฟลเดอร์ subdomain แล้วใช้ `./` ได้เลย ถ้าเป็นบัญชีหลักให้ใส่ path เต็ม เช่น `/domains/omnoigroup-it.com/public_html/lms/`
- ถ้า `ftps` ต่อไม่ติด ให้เปลี่ยน `protocol:` เป็น `ftp`
- **ระวัง:** ถ้า `api/config.php` บนเซิร์ฟเวอร์ถูกแก้ให้เป็นค่าจริงแล้ว อย่า push ไฟล์ `config.php` เวอร์ชัน placeholder ทับขึ้นไปโดยไม่ตั้งใจ — แนะนำเพิ่มเข้า `.gitignore` ตามหัวข้อด้านบน หรือใช้ `exclude` ใน workflow ไม่ให้ sync ไฟล์นี้ทับ

GitHub ใช้สำหรับ **เก็บประวัติเวอร์ชัน + auto-deploy** ส่วนวิดีโออยู่บน Google Drive และข้อมูลผู้ใช้/คะแนนอยู่บน MySQL ของ HostNeverDie

## ความปลอดภัยที่มีอยู่ในระบบ

- รหัสผ่านเก็บด้วย `password_hash()` (bcrypt) เท่านั้น ไม่มีการเก็บ plaintext ในฐานข้อมูล
- ทุกคำสั่ง SQL ผ่าน PDO prepared statements ป้องกัน SQL injection
- Session cookie เป็น `HttpOnly` + `SameSite=Lax` (+ `Secure` อัตโนมัติเมื่อเข้าผ่าน HTTPS) และสร้าง session ใหม่ทุกครั้งหลัง login สำเร็จ (ป้องกัน session fixation)
- ทุก endpoint ที่แก้ไขข้อมูล (POST) ต้องแนบ CSRF token ที่ตรงกับ session ปัจจุบัน
- ล็อกบัญชีชั่วคราว 15 นาทีหลัง login ผิดติดกัน 8 ครั้ง
- ข้อความที่แอดมิน/ผู้ใช้กรอก (ชื่อคอร์ส, ชื่อแผนก, ชื่อผู้ใช้ ฯลฯ) ถูก escape ก่อนแสดงผลเสมอ (กัน stored XSS)
- `.htaccess` ป้องกันการเข้าถึง `config.php`/`_bootstrap.php` ผ่าน URL ตรงๆ
- รหัสผ่านที่แอดมินตั้ง/รีเซ็ตให้พนักงาน จะแสดงเป็น plaintext **เพียงครั้งเดียว** ทันทีหลังบันทึก (เพราะระบบยังไม่มีอีเมลส่งอัตโนมัติ) — แอดมินต้องคัดลอกไปแจ้งพนักงานเอง ระบบจะไม่แสดงซ้ำอีก

## Migration สำหรับระบบที่ deploy อยู่แล้ว (อัปเดตนี้เพิ่มฟีเจอร์แก้ไข/ซ่อน-เผยแพร่คอร์ส + จัดการแบบทดสอบ)

อัปเดตนี้เพิ่มคอลัมน์ `courses.published` ใหม่ — ถ้าฐานข้อมูล production import `schema.sql` ไปก่อนหน้านี้แล้ว **ต้องรัน `migration_2026-07-17_course_publish.sql` ก่อน** ถึงจะอัปโหลดไฟล์ชุดใหม่ขึ้นเว็บได้ (ไม่งั้น API จะ error เพราะคอลัมน์นี้ยังไม่มีในตาราง):

1. DirectAdmin → MySQL Management → phpMyAdmin → เลือกฐานข้อมูล production
2. แท็บ **SQL** → วางเนื้อหาไฟล์ `migration_2026-07-17_course_publish.sql` → กด Go
3. ตรวจว่าตาราง `courses` มีคอลัมน์ `published` แล้ว (ค่าคอร์สเดิมทุกคอร์สจะเป็น `1` อัตโนมัติ — พนักงานยังเห็นคอร์สเดิมครบเหมือนก่อนอัปเดต ไม่มีคอร์สไหนถูกซ่อนโดยไม่ตั้งใจ)
4. จากนั้นค่อยอัปโหลด/deploy ไฟล์เว็บชุดใหม่ตามปกติ (ฐานข้อมูลใหม่ที่เพิ่ง import `schema.sql` เวอร์ชันล่าสุด ไม่ต้องรัน migration นี้ — คอลัมน์มีอยู่แล้วตั้งแต่ต้น)
